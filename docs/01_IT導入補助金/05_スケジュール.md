## 年間スケジュールイメージ

```
【1月〜2月】事前準備期間
  ├─ gBizIDプライム取得
  ├─ SECURITY ACTION自己宣言
  └─ IT導入支援事業者との事前相談

【3月】公募開始
  ├─ 公募要領公開
  ├─ IT導入支援事業者登録開始
  └─ 交付申請受付開始

【4月〜11月】申請・事業実施期間
  ├─ 1次〜最終締切まで複数回の申請機会
  ├─ 交付決定後、事業実施
  └─ 実績報告

【12月〜翌年1月】年度末処理
  ├─ 最終実績報告
  └─ 補助金交付
```

## 実績報告期限の考え方

実績報告期限は、交付決定日によって異なります。

### 計算例

```
交付決定日: 2026年5月25日
事業実施期間: 交付決定日〜2026年10月31日（例）
実績報告期限: 事業実施期間終了日から30日以内（2026年11月30日）
```

> **重要**: 実績報告期限を過ぎると補助金が交付されません。必ずスケジュール管理を行ってください。

## 効果報告スケジュール

効果報告は、補助金交付後、**毎年1回**（3年間または5年間）実施が必要です。

### 効果報告の時期

| 報告回 | 報告対象期間        | 報告時期                |
| ------ | ------------------- | ----------------------- |
| 1年目  | 事業実施完了〜1年後 | 対象期間終了後1ヶ月以内 |
| 2年目  | 1年目終了〜2年後    | 対象期間終了後1ヶ月以内 |
| 3年目  | 2年目終了〜3年後    | 対象期間終了後1ヶ月以内 |

> **注意**: 効果報告を怠ると、次回以降のIT導入補助金申請に影響があります。

## システムメンテナンス情報

申請システムは定期的にメンテナンスが行われます。

### 過去のメンテナンス例

- GビズIDメンテナンス: 月1〜2回、主に夜間（20:00〜23:00頃）
- 申請マイページメンテナンス: 不定期

> **注意**: 締切直前のメンテナンスで申請できなくなるリスクがあります。余裕を持った申請を推奨。
