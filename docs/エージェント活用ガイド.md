# 補助金コンサルエージェント 活用ガイド

> シーン別のプロンプト集と業務フローの実践ガイドです。

---

## クイックスタート

### 基本的な呼び出し方

```
@subsidy-consultant [質問・依頼]
```

オーケストレーターが適切なサブエージェントに委譲します。

---

## エージェント早見表

| 質問内容                   | 使うエージェント       |
| -------------------------- | ---------------------- |
| 対象者要件の診断           | `@eligibility-checker` |
| 最適な補助金の選定         | `@subsidy-selector`    |
| 申請フロー・必要書類       | `@application-guide`   |
| 締切・スケジュール         | `@schedule-tracker`    |
| 最新情報の調査             | `@web-researcher`      |
| 回答の記録                 | `@answer-recorder`     |
| 回答の品質チェック         | `@answer-reviewer`     |
| 複合的な相談（自動振分け） | `@subsidy-consultant`  |

## シーン別プロンプト集

### シーン1: 新規顧客からの問い合わせ対応

#### 「補助金を使いたいんですが、うちは対象になりますか？」

```
@eligibility-checker
以下の事業者が補助金対象になるか診断してください。

- 業種: 製造業
- 資本金: 5,000万円
- 従業員数: 45人
- 大企業の資本参加: なし
```

#### 「どの補助金がいいですか？」

```
@subsidy-selector
以下の導入を検討しています。最適な補助金を教えてください。

- 導入したいもの: 会計ソフトとPCを5台
- 目的: インボイス制度への対応
- 予算: 約200万円
- 事業者規模: 従業員8人の小売業
```

---

### シーン2: 提案書作成

#### 顧客向け提案書を作成

```
@subsidy-consultant
以下の顧客に補助金活用提案書を作成してください。

## 顧客情報
- 会社名: 株式会社サンプル製作所
- 業種: 製造業
- 従業員数: 30人
- 資本金: 3,000万円

## 導入意向
- 導入したいもの: 在庫管理システム
- 解決したい課題: 手作業での在庫管理に時間がかかっている
- 予算: 300万円程度
- 希望導入時期: 2026年10月まで
```

---

### シーン3: 申請サポート

#### 申請フローを説明してほしい

```
@application-guide
IT導入補助金のインボイス枠について、申請フローを詳しく教えてください。
法人での申請を予定しています。
```

#### 必要書類を確認したい

```
@application-guide
ものづくり補助金の申請に必要な書類を一覧で教えてください。
加点項目も取りたいので、その書類も含めてください。
```

---

### シーン4: スケジュール管理

#### 締切から逆算したスケジュールを作成

```
@schedule-tracker
IT導入補助金の1次締切（2026年4月21日）に向けて、
準備スケジュールを作成してください。

現在の状況:
- gBizID: 未取得
- SECURITY ACTION: 未宣言
- IT導入支援事業者: 未選定
```

#### 複数顧客のスケジュール管理

```
@schedule-tracker
以下の顧客の申請スケジュールを整理してください。

1. A社: IT導入補助金（2次締切希望）、gBizID取得済み
2. B社: ものづくり補助金（22次）、申請書作成中
3. C社: 持続化補助金、商工会議所への相談済み
```

---

### シーン5: 最新情報の確認

#### 公募要領の更新をチェック

```
@web-researcher
IT導入補助金の公式サイトで、最新の公募要領や
スケジュール更新がないか確認してください。
```

#### 採択結果を確認

```
@web-researcher
ものづくり補助金の21次締切の採択結果を確認してください。
採択率も教えてください。
```

---

### シーン6: 補助金の比較検討

#### どの補助金が最適か比較

```
@subsidy-selector
以下のケースで使える補助金を比較してください。

- 導入したいもの: 配膳ロボット3台
- 業種: 飲食業
- 従業員数: 15人
- 目的: 人手不足の解消

省力化投資補助金とIT導入補助金、どちらが適していますか？
```

#### 併用の可能性を確認

```
@subsidy-consultant
以下の投資を計画しています。複数の補助金を併用できますか？

1. 会計ソフト導入（50万円）
2. 製造ライン自動化設備（2,000万円）
3. ホームページリニューアル（30万円）
```

---

### シーン7: トラブル対応

#### 不採択になった場合の対応

```
@subsidy-consultant
IT導入補助金で不採択になりました。
次回申請に向けて、改善ポイントを教えてください。

不採択理由（推測）:
- 事業計画が抽象的だった可能性
- 導入効果の数値化が不十分
```

#### 交付決定前に契約してしまった

```
@application-guide
顧客が交付決定前にITツールの契約をしてしまいました。
どう対応すればよいですか？救済措置はありますか？
```

---

### シーン8: 回答の記録と品質管理

#### 回答を記録する

```
@answer-recorder
上記の回答を記録してください。
```

オーケストレーター経由で自動記録する場合：

```
@subsidy-consultant この回答を記録して
```

#### 記録済み回答の品質チェック

```
@answer-reviewer
Answer/2026-01/ の回答を品質チェックしてください。
```

## 業務フロー別の活用例

### 新規顧客対応フロー

```
1. @eligibility-checker で対象者診断
      ↓
2. @subsidy-selector で補助金選定
      ↓
3. @subsidy-consultant で提案書作成
      ↓
4. @schedule-tracker でスケジュール作成
      ↓
5. @answer-recorder で回答記録 → @answer-reviewer で品質チェック
```

### 申請サポートフロー

```
1. @application-guide で申請フロー確認
      ↓
2. @application-guide で必要書類確認
      ↓
3. @schedule-tracker で準備スケジュール作成
      ↓
4. @web-researcher で最新の締切確認
```

### 定期チェックフロー（週次）

```
1. @web-researcher で各補助金の更新確認
      ↓
2. @schedule-tracker で担当顧客の期限チェック
      ↓
3. リマインド対象の顧客に連絡
```

## 定期ワークフロー

### ナレッジベース更新（推奨: 週1回）

```
@subsidy-consultant ナレッジベースを最新情報に更新してください
```

公式サイトから最新情報を収集 → 差分分析 → 承認後更新 → ログ記録

### 顧客向けPPTX資料作成

```
@subsidy-consultant IT導入補助金の概要説明資料を10枚で作成してください
```

| 資料タイプ  | 用途                     | 例                                 |
| ----------- | ------------------------ | ---------------------------------- |
| A: 概要説明 | 補助金制度の汎用説明     | 「IT導入補助金の概要資料を作って」 |
| B: 個別提案 | 特定顧客向けカスタム提案 | 「A社向けに提案資料を作って」      |
| C: 比較資料 | 複数補助金の比較         | 「IT導入とものづくりの比較資料」   |

---

## 注意事項

1. **最新情報は必ず確認**: ナレッジベースは2026年1月時点の情報です
2. **公式サイトで最終確認**: 申請前に必ず公募要領を確認してください
3. **複雑なケースは専門家へ**: みなし大企業判定など複雑な判断は専門家に相談
