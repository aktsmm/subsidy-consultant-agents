# 回答品質チェックリスト

> **目的**: `answer-reviewer` エージェントが回答を評価する際の基準  
> **対象**: `/Answer` フォルダに記録されたすべての回答

---

## 品質評価基準

### 1. 正確性 (Accuracy)

| チェック項目 | 必須 | 説明 |
|-------------|:----:|------|
| 補助金名が正しい | ✅ | 正式名称を使用（略称は補足として記載） |
| 金額・補助率が正しい | ✅ | 最新の公募要領に基づく数値 |
| 申請要件が正しい | ✅ | 業種・規模等の条件が正確 |
| 締切日が正しい | ✅ | 最新スケジュールを反映 |
| 情報ソースが明記されている | ⚠️ | 公式サイト等の出典を記載 |

### 2. 完全性 (Completeness)

| チェック項目 | 必須 | 説明 |
|-------------|:----:|------|
| 質問に対して直接回答している | ✅ | 質問の意図を正確に把握 |
| 必要な前提条件を説明している | ✅ | 対象者要件等を明示 |
| 次のステップを提示している | ⚠️ | 具体的なアクションを提案 |
| 代替案・関連情報を提供している | ⚠️ | 該当しない場合の選択肢 |

### 3. 明瞭性 (Clarity)

| チェック項目 | 必須 | 説明 |
|-------------|:----:|------|
| 専門用語に説明がある | ⚠️ | 初心者にもわかる表現 |
| 構造化されている | ⚠️ | 見出し・箇条書きの活用 |
| 冗長でない | ⚠️ | 必要な情報を簡潔に |

### 4. 適時性 (Timeliness)

| チェック項目 | 必須 | 説明 |
|-------------|:----:|------|
| 情報が最新である | ✅ | 古い年度の情報でないこと |
| 締切に間に合う提案である | ⚠️ | 現実的なスケジュール感 |

## 評価スコア

### スコアリング方法

| 項目 | 配点 |
|------|------|
| 正確性 | 40点 |
| 完全性 | 30点 |
| 明瞭性 | 20点 |
| 適時性 | 10点 |
| **合計** | **100点** |

### 合格基準

| スコア | 判定 | アクション |
|-------|------|----------|
| 90〜100点 | ✅ 優秀 | 自動承認 |
| 70〜89点 | ⚠️ 合格 | 自動承認（改善提案付き） |
| 50〜69点 | ⚠️ 要改善 | 修正後に再レビュー |
| 0〜49点 | ❌ 不合格 | 再作成が必要 |

## 自動チェック項目

`answer-reviewer` が自動で検証する項目：

### 必須チェック

```yaml
auto_checks:
  - name: 補助金名の表記
    pattern: "(IT導入補助金|ものづくり補助金|小規模事業者持続化補助金|事業承継|省力化)"
    required: true
    
  - name: 年度の記載
    pattern: "20\\d{2}年度?"
    required: true
    
  - name: 情報ソースの記載
    pattern: "(https?://|公式サイト|公募要領)"
    required: false
    warning: "情報ソースの記載を推奨"
    
  - name: 注意喚起の有無
    pattern: "(注意|重要|※|⚠)"
    required: false
    info: "必要に応じて注意事項を記載"
```

### NGワードチェック

```yaml
ng_words:
  - pattern: "必ず(採択|交付)される"
    reason: "確約表現は禁止"
    
  - pattern: "100%"
    reason: "断定的な数値は避ける"
    
  - pattern: "絶対に"
    reason: "断定表現は避ける"
```

## よくある品質問題

### 1. 古い情報の引用

**問題**: 前年度の補助率・上限額を記載  
**対策**: 必ず `docs/` の最新情報を参照

### 2. 対象者要件の省略

**問題**: 「申請できます」と回答したが要件を説明していない  
**対策**: 業種・規模・必須準備（gBizID等）を必ず記載

### 3. 断定的な表現

**問題**: 「必ず採択されます」等の表現  
**対策**: 「採択される可能性があります」等の表現に修正

### 4. 次のアクション不明

**問題**: 情報提供のみで終わっている  
**対策**: 「次のステップ」セクションを追加
