## 📝 命名規則

### ファイル名

```
YYYYMMDD-HHMMSS_{topic}.md
```

| 部分       | 説明                   | 例                        |
| ---------- | ---------------------- | ------------------------- |
| `YYYYMMDD` | 日付（8桁）            | `20260128`                |
| `HHMMSS`   | 時刻（6桁）            | `143022`                  |
| `{topic}`  | トピック名（日本語OK） | `IT導入補助金_対象者診断` |

### フォルダ名

```
YYYY-MM/
```

月ごとにサブフォルダを作成します。

## 📋 テンプレート

各回答ファイルは以下の構造で記録されます：

```markdown
---
date: 2026-01-28T14:30:22
topic: IT導入補助金_対象者診断
status: draft | reviewed
agent: eligibility-checker
customer: N/A
---

## 質問

> （ユーザーからの質問原文）

## 回答

（エージェントの回答内容）

## メタデータ

- **回答エージェント**: eligibility-checker
- **補助金種別**: IT導入補助金
- **記録日時**: 2026-01-28T14:30:22

---

## レビュー結果

- [x] 根拠リンクあり
- [x] フォーマット準拠
- [x] 情報の整合性OK
- [x] メタデータ完備

**承認**: ✅ 自動承認済み
```

---

## 🔄 ワークフロー

```
回答完了 → answer-recorder → answer-reviewer → 承認済み
              │                    │
              ▼                    ▼
         Markdown作成         品質チェック
         /Answer/YYYY-MM/     status更新
```

### ステータス

| status     | 説明                 |
| ---------- | -------------------- |
| `draft`    | 記録済み、未レビュー |
| `reviewed` | レビュー済み、承認   |

## 🔍 検索・活用

### grep で検索

```bash
# 特定のトピックを検索
grep -r "IT導入補助金" Answer/

# 特定の顧客を検索
grep -r "customer: 株式会社ABC" Answer/
```

### VS Code で検索

1. `Ctrl+Shift+F` でワークスペース検索
2. `Answer/` フォルダに絞り込み
3. キーワードで検索

## ⚠️ 注意事項

1. **手動編集は非推奨**: ファイルはエージェントが自動生成します
2. **削除は慎重に**: 記録は業務ログとして保管されます
3. **機密情報に注意**: 顧客名や金額が含まれる場合があります
